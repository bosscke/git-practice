Git New Files

  • Tracked - files that Git knows about and are added to the repository
  • Untracked - files that are in your working directory, but not added to the repository
git status

Git Staging Environment

Staged files are files that are ready to be commited.

git add index.html

Using --all will stage all changes.

git add --all

Shorthand command is for add is . or -A.

git add -A

Git Commit

By adding clear messages to each commit, it is easy for you and others to see what has changed and when!

git commit -m "First release"

Check the status of the repo using --short option to see changes in a more compact way.

  • ?? - Untracked files
  • A - Files added to stage
  • M - Modified files
  • D - deleted files
git status --short

It is possible to commit changes directly, skipping the staging environment. The -a option will automatically stage every chnged already tracked file.

git commit -a -m "Update .. "

See the history of commits with log command.

git log

Git Branch

In Git, a branch is a new/separate version of the main repository. Branches allow you to work on different parts of a project without impacting the main branch.

git branch git-practice-2

To confirm that we created a new branch:

git branch

The * beside master indicates that we are currently still on the master branch. To move to the new created branch git-practice-2 use command checkout.

git checkout git-practice-2

To go back to master branch :

git checkout master

Create new branch to deal with a fix..:

git checkout -b emergency-fix

Git Branch Merge

Now we can merge the current branch (master) with emergency-fix. First make sure to be on master branch.

git checkout master

Now merge (master) and emergency-fix.

git merge emergency-fix

As master and emergency-fix are same, it can be deleted.

git branch -d emergency-fix
